# Implementation Plan: AAP Metrics Correlation Engine

**Branch**: `008-aap-metrics-correlation` | **Date**: 2026-03-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-aap-metrics-correlation/spec.md`

## Summary

Enhance the existing AAP correlation engine to use a six-tier fact-based matching hierarchy with explicit confidence scoring (0.0-1.0), asynchronous background execution, multi-tier reinforcement boosting, a reconciliation queue for weak matches, NOT_CORRELATED exclusion rules, full audit logging, and a temperature gauge UI component for visualising correlation confidence at both per-resource and fleet-aggregate levels.

This is an enhancement of existing code (`aap_correlation.py`, `aap_import.py`, `automations.py` endpoints, `AutomationReviewPage`), not a greenfield build.

## Technical Context

**Language/Version**: Python 3.12+ (backend), TypeScript 5.4+ (frontend)
**Primary Dependencies**: FastAPI, psycopg[binary] (v3, async), Pydantic v2, python-multipart; React 18, TanStack Query v5, Axios, lucide-react, Zustand
**Storage**: PostgreSQL 16 + Apache AGE extension (graph edges for AUTOMATED_BY; relational tables for reconciliation, exclusions, audit)
**Testing**: pytest (backend), manual integration testing (frontend)
**Target Platform**: Linux server (Docker container)
**Project Type**: Web application (FastAPI backend + React SPA frontend)
**Performance Goals**: 1,000 AAP hosts x 5,000 resources correlated in <30 seconds (SC-002); temperature gauge renders in <1 second (SC-005)
**Constraints**: No new external services (Constitution VIII); background tasks via FastAPI BackgroundTasks (no Celery/Redis)
**Scale/Scope**: Typical fleet: 1,000-5,000 resources, 500-2,000 AAP hosts
